# Product Guidelines - Contextware

## Communication Style
- **Technical & Precise:** Contextware uses a direct, concise, and technically accurate prose style. It focuses on clarity and efficiency, making it ideal for CLI environments and AI agent interactions.

## Project Personality
- **The Faithful Chronicler:** Contextware acts as a dedicated and reliable repository of project history. It systematically records decisions, context, and state, providing accurate information when explicitly queried.

## User Experience (UX) Principles
- **Zero-Configuration Tracking:** The system should operate seamlessly with minimal manual intervention. Context is gathered and organized automatically to ensure a frictionless experience.
- **Traceable & Accountable:** Every piece of retrieved information should be clearly linked to its source, whether it's a file, a past conversation, or a project decision.

## Documentation Standards
- **Semantic System Maps:** Focus on providing high-level summaries that explain the "why" behind components and files, creating a semantic map of the entire system.
- **Continuous Documentation:** Documentation should evolve alongside the code, with automatic updates reflecting changes in the project's state and architecture.
