# Project Design: Contextware (Skill)

## 1. Overview
**Contextware** is a Gemini CLI skill that provides persistent, semantic memory for software development. It enables the agent to remember project decisions, user preferences, and high-level code purpose across sessions.

## 2. Architecture
The system is managed via **`uv`** and runs entirely locally.

*   **Dependency Management:** `pyproject.toml` (managed by `uv`).
*   **Database:** `lancedb` (Local Vector DB).
*   **Embeddings:** `fastembed` (Local, CPU-friendly embeddings).
*   **Storage Path:** `skills/contextware/data/contextware_db/`.
*   **Execution:** All background tasks and scripts run via `uv run`.

## 3. Data Collections

### A. `facts` (The "Hippocampus")
*   **Model (`Fact`):**
    *   `content`: str (The fact or preference).
    *   `timestamp`: float (Creation time, default: `time.time()`).
    *   `vector`: Vector(384) (Generated from `content`).
*   **Source:** Explicitly saved by the agent using `store.py --type fact`.
*   **Content:** Concise statements (e.g., "User prefers Pytest").
*   **Used For:** Precise retrieval of constraints and preferences.

### B. `episodes` (The "Black Box")
*   **Model (`Episode`):**
    *   `goal`: str (The objective).
    *   `summary`: str (Narrative of actions and lessons).
    *   `result`: str (`success` | `failure` | `partial`).
    *   `category`: str (e.g., `shell`, `git`, `code`, `test`).
    *   `timestamp`: float (Creation time, default: `time.time()`).
    *   `vector`: Vector(384) (Generated from `goal` + `summary`).
*   **Source:** Explicitly saved by the agent to record outcomes of tasks.
*   **Content:**
    *   `goal`: The objective being pursued.
    *   `result`: `success` | `failure` | `partial`.
    *   `category`: `shell` | `git` | `code` | `test` | etc.
    *   `summary`: Narrative of actions taken and lessons learned.
*   **Used For:** Preventing infinite loops and repeating past mistakes.

### C. `code_index` (The "Map")
*   **Model (`CodeIndex`):**
    *   `file_path`: str (Absolute path, used for direct lookups).
    *   `summary`: str (High-level purpose of the file).
    *   `classes`: str (JSON-serialized map of class names to lists of member functions).
    *   `top_level_functions`: List[str] (List of functions defined outside classes).
    *   `last_modified`: float (File `mtime` at the time of indexing).
    *   `vector`: Vector(384) (Generated from `summary` + `classes` + `top_level_functions`).
*   **Source:** Generated by the agent or auto-extracted by `store.py` (for Python).
*   **Content:**
    *   `file_path`: Absolute path (Primary Key).
    *   `summary`: High-level purpose.
    *   `classes`: JSON map of classes to their methods.
    *   `top_level_functions`: List of top-level functions.
    *   `last_modified`: File mtime for freshness checking.
*   **Used For:** Semantic search for files and structural overview.

## 4. Workflows

### Memory Capture
1.  **Fact Capture:** The agent identifies a persistent preference or constraint and executes `uv run skills/contextware/scripts/store.py --type fact --content "<fact>"`.
2.  **Episode Capture (Black Box):** When a logical task or significant command finishes, the agent records the outcome:
    `uv run skills/contextware/scripts/store.py --type episode --goal "..." --result "success|failure" --category "..." --content "Summary of actions taken"`.
3.  **Storage:** `store.py` embeds the content (including metadata for episodes) and saves it to the relevant collection.

### Codebase Indexing
1.  The agent identifies a file that needs understanding or has been modified.
2.  If it's a Python file, the agent can rely on auto-extraction:
    `uv run skills/contextware/scripts/store.py --type index --path "path/to/file.py" --content "Detailed summary"`
3.  For other languages, or for more control, the agent provides symbols manually:
    `uv run skills/contextware/scripts/store.py --type index --path "path/to/file" --classes '{"ClassName": ["method1"]}' --functions "func1" --content "Detailed summary"`
4.  `store.py` calculates the embedding of the summary and symbols, records the current `mtime`, and updates the `code_index` collection.

### Retrieval (RAG)
1.  User asks: "Where do we define the data models?" or agent checks: "Have I tried this fix before?"
2.  **Action:** Agent executes `uv run skills/contextware/scripts/recall.py --query "..." --scope "all"|"memory"|"episodes"|"code"`.
3.  The system:
    *   Searches relevant collections via vector similarity.
    *   **For Episodes:** Returns structured records including `[RESULT] [CATEGORY] Goal: ... | Summary: ...`.
    *   **For Code:** Returns the path, summary, and symbols. It also checks `mtime` and prefixes with `[STALE]` if the file changed since indexing.
4.  **Metadata Lookup:** The agent can check a specific file's status using `uv run skills/contextware/scripts/recall.py --path "path/to/file"`.
5.  **Agent Action:** If info is missing or stale, the agent falls back to its native `read_file` tool to get full content and then re-indexes the file.

## 5. Scripts (CLI Interface)

The agent interacts with the Contextware skill by executing scripts via `run_shell_command`:

1.  **Store Data:** `uv run skills/contextware/scripts/store.py --type fact|episode|index [...]`
    *   Use `--delete` with the query/path to remove entries.
2.  **Recall Data:** `uv run skills/contextware/scripts/recall.py --query "<query>" [--scope S] [--limit L]`
    *   Use `--path <path>` for direct metadata lookup and staleness check.
